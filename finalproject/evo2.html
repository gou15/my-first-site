It's day 3 and, while it's not code (it's markup language), we start learning HTML and its structure. Lucky for all of us we can develop something a little nicer than that first page. Using element tags and inserting our content between those tags we learned how to change the way things are presented to the world. By assigning attributes to those tags we can give our document a little bit of spice. It's kind of cool.

The first 6 lines of this page are headers 1 thru 6. The block of text above this one (and this one) are paragraphs. They're inside a body tag which are inside the master HTML tag. The body tag is a sibling to the head tag which contains our information that isn't rendered to the user. We'll talk more about the HTML head later.

Sometimes you want to render some information to the user that your browser would otherwise interpret as HTML. If I want to show you a paragraph tag (which is a less than sign, a lower case "p", and then a greater than sign), how would I do that? If I type it out like normal:

your browser is just going to start a new paragraph.

You use something called HTML entities to force browsers to render characters that are normally rendered as HTML. These start with an ampersand and end with a semicolon which is the browser's flag to just render the data and not do anything special with it. Maybe you're explaining a new topic to someone and they need some definitions. The description definition list will cut it for you:

Savestate
An emulator tool that allows you to save at any time. Want to save after you just made that really hard jump but haven't made the checkpoint yet? Savestates let you do that.
Object
Things that Mario can interact but do not typically interact with Mario. Ground, blocks, water... these are objects. There are exceptions to this, but this definition is good enough for us beginners.
Sprite
Things that Mario can interact with and they can interact with him. P-switches, enemies, Bowser, Koopa shells, powerups... these are sprites.
Hitbox
The hitbox is the area of the sprite that detects whether or not the player or enemy has been hit. For instance, small Mario's sprite is almost twice the size of his hitbox.
