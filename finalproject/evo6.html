Our First Framework... Bootstrap Makes CSS A Breeze


I am not going to attempt to do a better job of explaining Bootstrap than Bootstrap can. Read the documentation. All I'm going to do with this page is show you what we learned in class, but there is so much you can do with this framework it's incredible. You're essentially importing a whole lot of CSS that a whole lot of other people worked on so you don't have to worry about it. You apply their hardwork by designating the correct classes to the correct HTML tags and all sorts of magic will happen for you. The documentation is so thorough that you should be able to copy and paste most of the "coding" and then modify what you need to modify to accomplish what you want. You can mix and match Bootstrap classes within the same tag, and there is nothing stopping you from importing your own CSS as well.

A thorough understanding of the box model is crucial to the application of Bootstrap styling. Check out the last evolution to study the box model further if you need to. Bootstrap makes laying out your boxes easier. It also has built in functionality to modify your layout for varying viewport sizes (desktop, laptop, table, cellphone). This paragraph and the heading above it have not been styled except by the global styling imported by Bootstrap. Going forward on this page we're going to exclusively use Bootstrap styling and none of our own CSS (except for borders so you can see the boundaries of the containers) Containers have green borders, rows have blue borders, and columns have red borders. Let's go.


This paragraph is in the first column of a row contained in a container. This "cell" has no defined with so will default to 100% width (12 column) if there are no other columns in this row. But there are other columns, also with no defined width. These columns will share space equally and total a width of 12 columns. Since there are a total of 3 columns in this row, each will automatically occupy 4 columns of space.

This paragraph is in the second column of the first row. It doesn't matter that this paragraph has less content than the first - because they occupy the same row the size of this container will default to the biggest cell.

This paragraph is in the third and final column of the first row.

This paragraph is the first in a new row which contains columns of defined width. No matter how many other columns are in this row, this particular cell will occupy 4 columns of space because that's the defined width of the column.

This paragraph is also in a column of defined width - also 4. Because the next column (even though it's in the same row) has a definied width of 8 columns it exceeds the max allowable columns per row, which is 12. It will automatically move itself to show underneath these first 2 cells. The cell to the right of this one is "empty" since we are trying to put too many columns on one row. It has no red border so there is no block element there.

This is the 8 width column I was talking about. Notice the lack of a blue border around this one since it is technically inside the same row as the the cells above it. Bootstrap will auto format your block elements based on the max column width. Note that as you zoom in and out or view this site on different sizes of devices it's layout will automatically adapt to the viewport of that device. Pretty cool eh? Utilizing this framework gives you a leg up on ensuring that your website is a pleasant browsing experience regardless of the device it is being viewed on.

I put these here as a 4 column width cell just to round off the block and demonstrate where these cells begin and end with the border legend per above.


The next container is set up with "breakpoints" to tell the browser what to do with each cell as the size of the viewport changes. The above container kept the same number of columns regardless of viewport. You'll start to see things moving around as you change your viewport size with the next container... oh hey your navbar followed you down the page! It's set to sticky, but you can do what you want with it. Consult the documentation!
